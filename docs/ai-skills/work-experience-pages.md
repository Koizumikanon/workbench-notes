下面是将已验证工作转化为职业作品集文档的通用模板。实际工作环境、客户和内部证据应保留在 private 记录中。

````markdown
---
name: work-experience-pages
description: Turn verified work requests, incident resolutions, project milestones, and historical work examples into private experience records and sanitized public documentation drafts. Use after a task is solved, validated, completed, or supplied as a past work case.
---

# Work Experience Pages

1. Wait until the result is verified; do not interrupt active troubleshooting to write a case study.
2. Record the original request, constraints, steps, validation, limitations, and reuse value in a private record.
3. Search existing public documentation before creating a new page.
4. Add a command reference for single-tool knowledge, a recipe for repeatable workflows, or an anonymous case for diagnostic and decision-making experience.
5. Generalize all environment details. Keep only the problem pattern, method, validation, and reusable conclusion.
6. Add visible metadata: last updated date, document type, and keywords.
7. Run public-document review before any external publication.
````

优先补充已有页面，避免把同一经验拆成许多难以检索的短文。
