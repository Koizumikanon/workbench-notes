# DeepSeek 命令行接入

> 最后更新：2026-07-10 | 类型：工具接入指南
>
> 关键词：`deepseek`、`cli`、`api`、`coding-agent`

DeepSeek 的命令行体验取决于你实际选择的客户端或 IDE 集成。不要假设所有环境都有一个统一的 `deepseek` 命令；先确认正在使用的软件、来源和版本。

## 先确认客户端

```bash
command -v deepseek
<client-command> --version
<client-command> --help
```

记录客户端名称、版本、安装来源、模型/服务端地址、配置文件位置和权限模式。涉及 API key 的配置只放在本机安全存储或 ignored 文件中，不能提交到仓库。

## 接入检查

1. 在空目录或测试仓库验证登录、模型调用和文件权限。
2. 确认是否支持项目规则、MCP、skills、会话恢复和审批模式。
3. 先以只读或计划模式处理真实项目。
4. 将验证过的客户端名称和日常命令补充到本页；不要把不确定的第三方命令当作通用标准。

## 何时补充本文档

确定一个具体客户端并在实际项目中跑通后，记录可复制的启动、恢复、权限和诊断命令，再更新本页的最后更新与关键词。
